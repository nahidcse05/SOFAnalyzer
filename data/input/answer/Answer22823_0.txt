<p>Navigate to the <a href="http://packages.ubuntu.com/aptitude" rel="nofollow">aptitude package page</a> on the <a href="http://packages.ubuntu.com/" rel="nofollow">Ubuntu website</a> (replace <code>ubuntu.com</code> by <code>debian.org</code> if you're on Debian). Pick the right version (<a href="http://packages.ubuntu.com/lucid/aptitude" rel="nofollow">lucid</a>) for you. You either need matching, working <code>aptitude</code> and <a href="http://packages.ubuntu.com/lucid/libapt-pkg-libc6.10-6-4.8" rel="nofollow"><code>libapt-pkg-*</code> (provided by <code>apt</code>)</a> packages. Follow the download link for your architecture and download the <code>.deb</code> file(s) that you need.</p>

<p>You can also navigate to the package archive on a <a href="https://wiki.ubuntu.com/Mirrors" rel="nofollow">Ubuntu mirror</a> (<a href="http://www.debian.org/mirror/list" rel="nofollow">Debian mirror</a>) near you. You can find out precisely which package sources your system normally uses by looking in <code>/etc/apt/sources.list</code> and <code>/etc/apt/sources.list.d/*.list</code> on your system. The packages are organized by distribution component (main/restricted/universe/multiverse for Ubuntu, main/contrib/non-free for Debian) and then by source package. For example, look in <a href="http://us.archive.ubuntu.com/ubuntu/pool/main/a/apt/" rel="nofollow">http://us.archive.ubuntu.com/ubuntu/pool/main/a/apt/</a> . All the versions carried by the mirror are in the same directory, so make sure you pick the right one for your system. Note that security upgrades <a href="http://unix.stackexchange.com/questions/4255/why-is-there-a-separate-package-repository-for-debian-security-updates">live on a separate server</a>.</p>

<p>Once you've downloaded the <code>.deb</code> files, install them by running the <code>dpkg</code> command as root:</p>

<pre><code>sudo dpkg -i /path/to/package.deb
</code></pre>
